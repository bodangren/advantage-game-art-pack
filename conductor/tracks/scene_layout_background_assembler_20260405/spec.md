# Specification

## Overview

Build the deterministic `background_scene` assembler. This compiler must create
full scene backgrounds from approved tiles, props, motifs, and layout
templates. It is the path for assets like `library_background.png`,
`background.png`, and future mini-game backdrops.

The assembler must preserve the series style while also reserving gameplay
space so the resulting backgrounds remain practical for the mini-games.

## Functional Requirements

### FR1: Scene Manifest Schema

- Define a strict manifest schema for `background_scene` programs.
- The schema must support:
  - canvas size
  - theme and subtheme
  - layout template
  - reserved gameplay zones
  - tile sources
  - prop placement groups
  - focal motifs
  - lighting zones
  - decal passes
  - output variants

### FR2: Layout Templates

- Provide deterministic layout templates for at least:
  - `library_room`
  - `ruins_courtyard`
- Templates must define major spatial zones, camera angle assumptions, and
  reserved negative space.

### FR3: Tile and Prop Assembly

- Assemble backgrounds from approved tiles and props rather than painting them
  ad hoc.
- Placement rules must support:
  - wall/floor partitioning
  - shelf and table placement
  - clutter clusters
  - path or stage areas
  - symmetry or weighted asymmetry depending on template rules

### FR4: Lighting and Decal Passes

- Add deterministic scene passes for:
  - global lighting direction
  - local emissive sources
  - contact shadows
  - ground decals such as cracks, grass, dust, or runes
- Each pass must remain bounded by the style canon.

### FR5: Scene Manifest Export

- Export both the final image and a machine-readable placement manifest.
- The placement manifest must include tile references, prop references, and
  bounding boxes for reserved gameplay zones.

## Non-Functional Requirements

- Identical scene manifests must render identical images and placement manifests.
- A debug mode must be available so implementers can inspect zones and
  placements visually.
- Layout logic must stay deterministic even when multiple valid placements exist.
- Scene outputs and manifests must be ingestible by the candidate-generation and
  critic-loop tracks without manual translation.

## Deliverables

- `background_scene` schema
- Layout template files
- Scene assembler modules
- Placement/debug overlay export
- Sample scene manifests for library and ruins themes
- Tests for layout resolution, placement, and export integrity

## Acceptance Criteria

- A library scene manifest renders deterministically at the target resolution.
- A ruins scene manifest renders deterministically at the target resolution.
- Both sample scenes export placement manifests that include reserved gameplay
  zones.
- The sample scene outputs can be handed to the candidate-generation and
  critic-loop flow without rewriting their manifests by hand.
- Debug overlays make major zones and placements understandable to a human.
- No scene is generated by freehand drawing logic that bypasses approved tiles,
  props, and motifs.

## Out of Scope

- Review UI
- LLM prompt planning
- Auto-approval policy
- High-volume batch orchestration
